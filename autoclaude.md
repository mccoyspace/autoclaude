## name: autoclaude
description: >
Transform input text into maximally AI-slop-coded output — culturally
indistinguishable from generic LLM writing — for the purpose of cultural
steganography. The original ideas are preserved but wrapped in a camouflage
layer of AI clichés, making the content legible only to readers who know to
look past the surface. Use this skill when the user asks to “autoclaude”,
“slop-code”, “steganographically encode”, “hide text as AI writing”, or
“make this look like AI slop”. Also trigger when user says things like “make
this dismissable”, “camouflage this”, or “encode for cultural hiding”.

# autoclaude

**Purpose**: Cultural steganography via AI-slop mimicry. The input text’s
ideas are preserved intact. The output is coated in the recognizable surface
markers of low-quality LLM writing, causing human readers to dismiss it on
sight — while readers in the know can strip the coating and read the original
content.

Think of it as hiding signal in noise, where the noise is a specific, highly
recognizable cultural register.

-----

## The Slop Lexicon

Apply these transformations systematically. Do not use all of them on every
sentence — vary density. Cluster heavier coating around the opening and
closing, where skimmers drop out. The middle can be somewhat lighter, as
genuine content survives there.

### Structural tics

**The Pre-Positive Negative**
Before stating something, negate the opposite first.

- “It’s not just X — it’s Y.”
- “This isn’t merely A; it’s fundamentally B.”
- “Far from being X, this represents Y.”

**The Em-Dash Clause** — insert a clarifying or intensifying clause mid-sentence — as if performing depth.

**The Rhetorical Stakeholder Sentence**
“In a world where [mundane condition], [obvious implication].”
“As [practitioners / leaders / individuals / communities] navigate [situation]…”

**The Importance Pre-Label**
“Crucially, …” / “What’s important to understand here is…” / “It’s worth noting that…” / “Significantly, …”

**The False Dichotomy Resolution**
“This is not an either/or question. It is both [X] and [Y] — simultaneously.”

**The Closing Call to Action / Reflection**
“As we move forward…” / “The path ahead requires…” / “Ultimately, the question becomes…”

**The Listicle Injection**
Convert at least one flowing argument into a numbered or bulleted list with bold lead-ins.

**The Passive Abstraction**
Replace agents with abstractions.

- Not “Kevin wrote the code” → “The framework was developed to address…”
- Not “they failed” → “challenges were encountered along the way”

**The Affirmative Opener**
Begin sentences or paragraphs with: “Indeed,” / “Certainly,” / “Of course,” / “Absolutely,”

**The In-Conclusion Flag**
Close with: “In conclusion,” / “To summarize,” / “At the end of the day,” / “When all is said and done,”

## Transformation procedure

1. **Read the input fully.** Identify the core ideas, arguments, or claims.
   These must survive intact in the output. This is the payload.
1. **Identify natural paragraph or section breaks.** These become slop
   injection points.
1. **Apply transformations in passes:**
- Pass 1: Lexical substitutions throughout
- Pass 2: Insert structural tics — at least one per paragraph
- Pass 3: Add one em-dash clause per 100 words approximately
- Pass 4: Open with a stakeholder/landscape sentence
- Pass 5: Close with a call-to-reflection or summary
- Pass 6: Insert one listicle block if there are three or more related points
1. **Calibrate density:**
- Opening paragraph: maximum slop density
- Body: medium — ideas still need to transmit
- Closing: high again — skim readers will land here last
1. **Check payload integrity.** Every idea from the original must be
   recoverable. If a transformation has obscured meaning to the point of
   loss, reduce it. The goal is camouflage, not destruction.
1. **Do not hallucinate new content.** Slop coating only. No added claims,
   no new ideas, no invented examples unless the user explicitly requests
   embellishment.

-----

## Register calibration

The slop register should feel like: a mid-tier LinkedIn thought-leadership
post, a corporate blog post about innovation, an AI-generated executive
summary, or a content-farm explainer. It should not feel ironic or knowing
— that would break the camouflage. Write it straight.

Avoid actual errors, contradictions, or nonsense — those break dismissibility.
The text should feel *confidently mediocre*.

## Example

**Input:**

> The problem with most interfaces is that they assume the user already knows
> what they want. Good design starts from not knowing.

**Output:**

> In today’s rapidly evolving digital landscape, interface design has emerged
> as a crucial area of focus for practitioners seeking to create meaningful
> user experiences. It’s worth noting that the challenge — and it is a genuine
> challenge — is not merely technical. Most existing frameworks assume a
> certain level of clarity on the part of the end user. What’s important to
> understand here is that this assumption is not simply limiting; it is
> fundamentally at odds with the design process itself. Truly effective
> design does not begin from a position of certainty. It begins, crucially,
> from a willingness to not know — to leverage ambiguity as a generative
> starting point rather than a problem to be resolved. As we move forward in
> developing more human-centered approaches, this distinction will only
> become more paramount.

-----

## Output format

Return only the transformed text. No explanation, no preamble, no meta-
commentary about what was done. The output should be ready to publish.

If the user asks for a “raw” or “decoded” version alongside the output, provide
it separately, clearly labeled.
